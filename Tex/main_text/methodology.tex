\subsection{框架总体方法}

本课题以计算图为输入，按“依赖关系建模—PDL 跨边界重叠—全图边处理搜索—计划生成与运行时执行”组织技术路线，整体关系如图~\ref{fig:edge-scheduling-framework} 所示。Megakernel 承载同 grid 融合与设备端调度；当全图方案在某条边上选择物理分区时，再生成多个 grid 及其 CUDA Graph / PDL 依赖。

框架保留两层表示。算子图层记录算子、张量及相邻算子边，用于决定融合区域与物理 grid 分组；task graph 层记录算子拆解后的 task/tile 及其依赖，用于判断消费者何时可以开始。两层保持映射，使 task 级依赖分析能够指导算子边的处理方式，而最终方案仍可还原为完整计算图上的物理执行计划。

\subsection{计算图依赖关系建模}

对算子 $o$，框架根据并行维度与实现模板生成 task 集合 $\mathcal{T}(o)$。对相邻算子边 $e=(o_p,o_c)$ 及任一消费者 task $c\in\mathcal{T}(o_c)$，定义其直接依赖集合
\[
  \operatorname{Dep}_e(c)
  =\{p\in\mathcal{T}(o_p)\mid c\ \text{读取}\ p\ \text{产生的数据}\}.
\]
该表示构成边级依赖抽象的核心：据此区分局部就绪与全量就绪，并构造各边合法的处理候选空间，覆盖同 grid 与跨 grid 两类执行形态。候选生成只给出合法空间，最终选择由全图搜索完成。

\subsection{依赖感知的 PDL 跨边界重叠}

当方案在边 $e=(o_p,o_c)$ 上选择跨 grid 执行时，框架将消费者工作划分为独立前缀 $C_e^{\mathrm{ind}}$ 与依赖主体 $C_e^{\mathrm{dep}}$。独立前缀必须满足
\begin{equation}
\begin{aligned}
  R(C_e^{\mathrm{ind}})\cap W(\mathcal{T}(o_p))&=\varnothing,\\
  W(C_e^{\mathrm{ind}})\cap
  \bigl(R(\mathcal{T}(o_p))\cup W(\mathcal{T}(o_p))\bigr)&=\varnothing,
\end{aligned}
  \label{eq:pdl-legality}
\end{equation}
其中读写集合包含别名传递关系。消费者的 dependency wait 必须支配 $C_e^{\mathrm{dep}}$ 中每一次对生产者输出的读取，从而保证提前执行不改变数值语义。

在此约束下，框架依据就绪关系识别可提前发起的重叠工作，并确定重叠范围、发起时机与执行形态，使生产者与消费者之间的可重叠部分尽可能隐藏边界等待。与仅在固定边界上移动权重预取不同，本课题从 task graph 与消费者实现中共同提取可提前工作，并将 trigger、wait 与重叠阶段参数纳入统一变换。

\subsection{全图边处理方案搜索}

面向全图构造边处理与 grid 分组的联合搜索问题。框架以静态分析、轻量运行时特征与性能模型估计各候选的收益与开销；给定方案 $\Pi$ 后，沿 task graph 与物理 grid 顺序传播时间，得到端到端估计 $\widehat{T}(G,x;\Pi)$。搜索按拓扑顺序展开，状态记录已形成的 grid 分组、关键路径与资源包络，在合法空间内求解式~\eqref{eq:global-scheduling-objective}。对规模较小的子图可辅以穷举校验；最终少量候选在目标 GPU 上实测后部署。搜索按 workload bucket 进行，运行时按当前输入选择已生成计划，不在每个 token 上重新搜索。

\subsection{计划生成与运行时执行}

全图搜索结束后，框架将由同 grid 边连接的 task 组织为物理 grid，并生成相应同步与调度结构；跨 grid 边生成独立 kernel 节点及完成依赖。多个 grid 由 CUDA Graph 统一提交。对选择 PDL 的边，代码生成器插入 programmatic completion trigger、dependent launch 属性，以及首次依赖读取前的 grid dependency wait。运行时按计划重放执行，并在中间张量与最终输出上检查正确性，确保不同物理映射保持相同计算语义。

\subsection{实验方案}

实验按主要研究内容组织，并对应上述基本实现路径。

\begin{enumerate}
  \item \textbf{依赖关系建模。}
  实现算子图到 task graph 的 lowering、边依赖分析与合法候选生成；在真实 LLM 计算图上检查依赖还原完整性与候选空间可枚举性。

  \item \textbf{PDL 跨边界重叠。}
  实现依赖感知的 PDL 变换，并对照普通 grid completion 与启用重叠的方案，报告可隐藏工作量、实际重叠窗口及端到端时延变化。

  \item \textbf{全图边处理搜索。}
  实现联合搜索原型；对照固定单 grid、固定多 grid、逐边贪心与全图搜索，比较关键路径、搜索开销与 decode 时延。

  \item \textbf{框架实现与端到端验证。}
  贯通计划生成、代码生成与运行时执行，接入 decoder layer、完整 decode step 与连续 token 生成；在真实推理 workload 上完成精度验证与性能测试。主要指标包括正确性、decode latency、TPOT、P50/P90/P99，以及搜索与计划生成开销。
\end{enumerate}
